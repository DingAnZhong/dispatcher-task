# task

```
Program Design Test：
请利用golang的协程特性和并发调度CSP模型完成供应与需求间的匹配系统设计：
具体场景如：骑手与订单的分配
随机构建100位不同的骑手（uid,位置），在30秒内聚集1w订单的突发情况下，系统在最短的时间内完成分配,分配可以先就近原则，请根据你的算法统计出分配订单最少的后10名骑手，并输出他们的订单数
为了验证程序的合理性：test需满足不同t（N：M），例如：1分钟内1千个骑手对应10w订单，3分钟内1w骑手对应100w订单，10分钟内10w骑手对应1000w订单
要求：在最短和最少的约束条件下，机器资源理论上是2核心4G内存
Reference：https://mp.weixin.qq.com/s/SeuttNPZo21xLuzHxoGg6w
```

## 思路

### 1、网格搜索

极端情况下100K骑手，10M订单需要计算1000B次，时间复杂度爆炸，如果先在网格中找到候选骑手，时间复杂度会依据网格划分下降100~1000倍
可能会损失一些精度，局部最小不一定是全局最小，但是在大量骑手的情况下，可以忽略

### 2、削峰队列

在极端时间内涌入大量订单，比如饭点到了。虽然业务层处理很快，但是DB层承受不了这么高的TPS。需要一个准备队列削峰，并且在各次重试中顶部一直是高优先级（分配订单目前不考虑失败重试）。
